## 5. 進め方ルール

1. 大規模リファクタは段階分割: 1コミット = 1テーマに絞る
2. コミット前に動作確認: `npm run dev` で開発サーバ確認 → 私のスクショ確認後に進める
3. エラー時は git reset: 推測修正で深掘りせず、迷ったら一度戻す
4. ブランチは `feature/` プレフィックス: 例 `feature/matrix-table-expansion`
5. PR 本文は日本語で構造化: 概要 / 変更点 / 維持要素 / 動作確認 / 後続予定 / コミット一覧
6. マージ後はローカル後始末: `git checkout main && git pull && git branch -d <branch>`
7. 認証なし API ルートが既存: Phase 8 後の一括対応予定なので新規 API も同様で OK（後で揃える）
